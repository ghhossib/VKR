# SHD DRIVE

Веб-приложение каршеринга на Django + Django REST Framework.
Проект включает клиентский интерфейс бронирования, менеджерскую панель, чат по заявкам и REST API.

## Технологии

- Python
- Django 4.2
- Django REST Framework 3.16.1
- Pillow 10.4.0 (оптимизация загруженных изображений)
- SQLite (по умолчанию)

## Основной функционал

### 1) Витрина и каталог автомобилей

- Главная страница с лендингом и каталогом доступных автомобилей.
- Карточки показывают класс, год, коробку, количество мест, описание и цену за час.
- На клиенте реализован быстрый фильтр по классам (`all`, `business`, `suv`, `comfort`, `economy`).
- В каталог попадают только активные автомобили (`is_active=True`).

### 2) Аккаунт и авторизация

- Регистрация пользователя (логин, email, пароль).
- Вход/выход через стандартную auth-систему Django.
- После регистрации пользователь автоматически авторизуется.

### 3) Бронирование автомобиля

- Страница автомобиля позволяет создать бронирование в двух режимах:
- `quick`: быстрое бронирование от текущего времени +5 минут с выбором длительности (30 мин, 1/3/6/24 часа).
- `planned`: выбор одного из готовых 2-часовых слотов (формируются на ближайшие 16 часов).
- Валидации при создании:
- окончание должно быть позже начала;
- нельзя бронировать в прошлом;
- нельзя создать пересечение по времени для одной машины, если существующая бронь в статусе `created` или `confirmed`.
- После успешного создания бронь получает статус `created`.
- Пользователь может отменить бронь, если она в статусе `created` или `confirmed`.

### 4) Личный кабинет клиента

- Раздел «Мои бронирования»:
- список броней с периодом, датой создания и расчетной стоимостью;
- отображение текущего статуса;
- кнопка отмены (если статус позволяет);
- отображение причины отмены, если бронь отменена менеджером.

### 5) Чат по бронированию (клиент ↔ менеджер)

- Для каждой активной/подтвержденной брони доступен отдельный чат.
- Доступ к чату имеют только:
- владелец брони;
- сотрудники (`is_staff=True`).
- Сообщения сохраняются с автором и временем.
- Есть серверная пагинация (по 20 сообщений) и автообновление через AJAX каждые 5 секунд.

### 6) Менеджерская панель (`/dashboard/...`)

- Доступ только сотрудникам (`is_staff`).
- Управление бронированиями:
- список заявок с фильтрами по поиску, статусу и диапазону дат;
- пагинация (10 записей на страницу);
- действия по жизненному циклу брони:
- `created -> confirmed` (подтверждение);
- `created -> canceled` (отмена с обязательным комментарием менеджера);
- `confirmed -> completed` (завершение);
- переход в чат с клиентом.
- Ведется история смен статусов (`BookingStatusHistory`) с фиксацией:
- старого и нового статуса;
- кто изменил;
- комментария;
- времени изменения.
- Управление автопарком:
- добавление/редактирование/удаление автомобиля;
- включение/скрытие автомобиля через флаг `is_active`;
- загрузка фото.
- Управление пользователями:
- создание, редактирование, удаление пользователей;
- изменение пароля;
- назначение ролей:
- `client` (обычный пользователь),
- `manager` (`is_staff=True`),
- `admin` (`is_staff=True`, `is_superuser=True`);
- защита от удаления текущего авторизованного пользователя.

### 7) Работа с изображениями автомобилей

- При сохранении автомобиля загруженное изображение автоматически:
- корректируется по EXIF-ориентации;
- сжимается и ограничивается до `1600x1600`;
- конвертируется в `WEBP` (quality 78).
- Для отображения используется:
- локальный загруженный файл (`image_file`), если есть;
- иначе внешний URL (`image_url`).

## REST API

Базовый префикс: `/api/`

### Автомобили

- `GET /api/cars/` — список активных автомобилей.
- `GET /api/cars/?class=business` — фильтр по классу.

### Бронирования

- `GET /api/bookings/` — список только текущего авторизованного пользователя.
- `POST /api/bookings/` — создать бронь.
- `GET /api/bookings/{id}/` — получить свою бронь.
- `PUT/PATCH /api/bookings/{id}/` — изменить свою бронь (с той же валидацией интервалов).
- `DELETE /api/bookings/{id}/` — удалить свою бронь.

Ограничения API:

- `cars` доступны всем (`AllowAny`).
- `bookings` доступны только авторизованным (`IsAuthenticated`).
- Поле `status` у бронирования read-only на уровне сериализатора.
- `user` проставляется автоматически из `request.user`.

## Модели домена

- `Car` — автомобиль.
- `Booking` — бронирование.
- `BookingMessage` — сообщение в чате по брони.
- `BookingStatusHistory` — журнал смены статусов.

Статусы брони:

- `created` — создано;
- `confirmed` — подтверждено;
- `completed` — завершено;
- `canceled` — отменено.

## Тесты

В проекте есть автотесты (`cars/tests.py`), покрывающие ключевые сценарии:

- успешное создание брони через API;
- запрет пересекающихся интервалов;
- завершение брони менеджером с созданием записи в истории статусов;
- запрет доступа к чату чужой брони;
- фильтрация и пагинация списка бронирований в панели менеджера.

Логины и пароли:
админ = admin 123
менеджер = nik16 de5tnosu
клиент = 123 de5tnosu

Запуск:

```bash
python manage.py test
```

## Структура проекта

```text
shd_backend/      # настройки Django-проекта, root urls
cars/               # бизнес-логика, модели, формы, API, вьюхи, тесты
templates/cars/     # HTML-шаблоны клиентской и менеджерской частей
static/css/         # стили интерфейса
media/              # загружаемые файлы (фото автомобилей)
cars/fixtures/      # seed-данные
```

## Запуск проекта

```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py loaddata cars/fixtures/cars_seed.json
python manage.py createsuperuser
python manage.py runserver
```

После запуска:

- Главная: `http://127.0.0.1:8000/`
- API автомобилей: `http://127.0.0.1:8000/api/cars/`
- API бронирований: `http://127.0.0.1:8000/api/bookings/`
- Django Admin: `http://127.0.0.1:8000/admin/`
- Панель менеджера: `http://127.0.0.1:8000/dashboard/bookings/`
